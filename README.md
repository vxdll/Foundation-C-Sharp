<h2> First Steps </h2>

После установки .NET SDK и VS Code потренируемся с .NET CLI (набор консольных утилит).

1. Откроем cmd (у меня Windows)
2. Создадим каталог для проекта .Net: `mkdir dotnet`
3. Для перехода в каталог используем cd: `cd dotnet`
4. Базовая конструкция для консольного приложения, через `-o` добавим имя проекта helloworld
   `dotnet new console -o helloworld`
5. Перейдем в проект и запустим `program.cs` со скриптом:<br>
   `cd helloworld` <br>
   `dotnet run`
6. Вывод >> Hello, World! (Это база мой друг!)

Если физически открыть католог с данным проектом, и через IDE открыть Program.cs, то мы можем увидить исходный код и его легко изменить.

**Методы**

1. Console.Write() - метод вывод строки, без создания новой строки.
2. Console.ReadLine() - метод для чтения данных с консоли
3. var - создание переменной без явно указания типа данных, он определяется от содержимого переменной.

Немного практики

```csharp
Console.Write("What is you'r name: ");
var name = Console.ReadLine();
Console.WriteLine($"Hello {name}");
```

1. Запускаем `dotnet run`
2. Вводим своё имя (Моё Grisha)
3. Hello Grisha

Вывод данных через конструкцию $"{}" называется `интерполяция строк`.

<h2>Структура программы</h2>

Весь код программы на C# помещается в файл с раширением `.cs`
Именно код `program.cs` - выполняется по умолчанию

Любая программа состоит из строительных блоков - `инструкций` (statement).
Инструкция представлят собой некоторое действие, вызов метода, объявление переменной и так далее.
Оканчивается инструкция обязательно `;`

Инструкции можно объединять при помощи фигурных скобок:
``` csharp
{
	Console.WriteLine("Привет");
	Console.WriteLine("Хорошего тебе дня!");
}
```
В С# есть комментарии они не обрабатывются при компиляции.<br>
// - однострочный комментарий<br>
/* */ - мгногострочный комментарий

В каждом проекте С# есть файл, который отвечает за конфигурацию проекта. Имеет расширение `.csproj`
Файл представлен в виде xml кода и определяет конфигурацию.
Важно:
1. OutputType - тип проекта на выходе. Может быть всем известный `.exe` или `.dll` - некоторый набор функциональностей, который используется другими проектами.
2. TargetFramework - определяет версию фреймворка .NET. В нашем случае `net10.0`
